# V6 故障排查指南

## 🔍 常见问题及解决方案

### 问题 1：设置按钮点击无反应

**症状**：
- 点击扩展弹窗右上角的 ⚙️ 按钮
- 没有打开设置页面

**可能原因**：
1. 扩展未正确加载
2. manifest.json 配置错误
3. 浏览器权限问题

**解决方案**：

```bash
# 1. 检查浏览器控制台
# 打开扩展弹窗，按 F12，查看是否有错误

# 2. 重新加载扩展
# 访问 chrome://extensions/
# 找到扩展，点击"重新加载"

# 3. 检查 manifest.json
# 确认 web_accessible_resources 包含 settings.html
```

**验证**：
```javascript
// 在扩展弹窗的控制台中运行
chrome.runtime.getURL('settings.html')
// 应该返回：chrome-extension://[id]/settings.html
```

### 问题 2：API Key 保存失败

**症状**：
- 在设置页面输入 API Key
- 点击"保存设置"
- 显示"保存失败"

**可能原因**：
1. chrome.storage 权限问题
2. API Key 格式错误
3. 浏览器存储空间不足

**解决方案**：

```javascript
// 1. 检查 API Key 格式
// 正确格式：sk-xxxxxxxxxxxxxxxxxxxxxxxx
// 必须以 sk- 开头

// 2. 手动测试存储
chrome.storage.local.set({ test: 'value' }, () => {
  console.log('存储测试成功');
});

// 3. 检查存储空间
chrome.storage.local.getBytesInUse(null, (bytes) => {
  console.log('已使用存储空间:', bytes, '字节');
});
```

**验证**：
```javascript
// 在设置页面控制台运行
chrome.storage.local.get(['deepseekApiKey'], (result) => {
  console.log('API Key:', result.deepseekApiKey);
});
```

### 问题 3：API 连接测试失败

**症状**：
- 点击"测试连接"按钮
- 显示"连接失败"

**可能原因**：
1. API Key 无效或过期
2. 网络连接问题
3. VPN 阻止 API 请求
4. DeepSeek API 服务异常

**解决方案**：

```javascript
// 1. 手动测试 API
fetch('https://api.deepseek.com/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer sk-your-api-key'
  },
  body: JSON.stringify({
    model: 'deepseek-chat',
    messages: [{ role: 'user', content: '你好' }],
    max_tokens: 10
  })
})
.then(res => res.json())
.then(data => console.log('API 响应:', data))
.catch(err => console.error('API 错误:', err));

// 2. 检查网络
fetch('https://api.deepseek.com')
  .then(() => console.log('网络正常'))
  .catch(() => console.error('网络异常'));

// 3. 检查 VPN
// 尝试关闭 VPN 后再测试
```

**常见错误码**：
- `401 Unauthorized`: API Key 无效
- `429 Too Many Requests`: 请求过于频繁
- `500 Internal Server Error`: DeepSeek 服务异常
- `Network Error`: 网络连接问题

3. 简历内容不完整

**解决方案**：

```javascript
// 1. 查看 AI 原始响应
```

### 问题 6：自动填充不工作

**症状**：
- 点击"自动填充"按钮

**可能原因**：
1. 简历数据未保存
2. content.js 未注入
3. 表单字段无法识别
4. 网站使用特殊表单

**解决方案**：

```javascript

chrome.storage.local.get(['resume'], (result) => {
  console.log('简历数据:', result.resume);
});

// 2. 检查 content script
// 在目标网页按 F12，在控制台运行：
console.log('Content script 已加载');

// 3. 手动触发填充
chrome.runtime.sendMessage({ action: 'triggerAutofill' });

// 4. 检查表单字段
document.querySelectorAll('input, textarea, select').forEach(input => {
  console.log('字段:', input.name, input.id, input.placeholder);
});
```

**调试步骤**：
1. 打开招聘网站
2. 按 F12 打开控制台
3. 点击扩展的"自动填充"
4. 查看控制台输出

**预期日志**：

### 问题 7：扩展加载失败

**症状**：
- 在 chrome://extensions/ 加载扩展
- 显示错误信息

**可能原因**：
1. manifest.json 格式错误
2. 文件路径错误
3. 权限配置错误

**解决方案**：

```bash
# 1. 验证 manifest.json
# 使用 JSON 验证工具检查语法

# 2. 检查文件路径
# 确认所有引用的文件都存在：
# - popup-v4-local.html
# - popup-v6-ai.js

# - background.js
# - content.js
# - libs/pdf.min.js
```

**常见错误**：
- `Manifest file is missing or unreadable`: manifest.json 不存在或格式错误
- `Could not load icon`: 图标文件不存在
- `Invalid value for 'permissions'`: 权限配置错误

### 问题 8：VPN 环境下 API 调用失败

**症状**：
- 使用 VPN 时 API 调用失败
- 关闭 VPN 后正常

**可能原因**：
1. VPN 阻止 DeepSeek API 域名
2. VPN 导致网络延迟过高
3. VPN 修改了请求头

**解决方案**：

```javascript
// 1. 测试 API 可达性
fetch('https://api.deepseek.com')
  .then(() => console.log('API 可达'))
  .catch(() => console.error('API 不可达'));

// 2. 增加超时时间
const controller = new AbortController();
const timeoutId = setTimeout(() => controller.abort(), 30000); // 30秒

fetch('https://api.deepseek.com/v1/chat/completions', {
  signal: controller.signal,
  // ...
});

// 3. 使用代理（如果需要）
// 在 VPN 设置中添加 api.deepseek.com 到白名单
```

**建议**：
- 使用稳定的网络环境
- 如果 VPN 必须开启，配置 API 域名白名单
- 或者不配置 API Key，使用正则解析（离线可用）

### 问题 9：解析速度太慢

**症状**：
- AI 解析需要 10 秒以上
- 或者一直显示"正在解析"

**可能原因**：
1. 网络延迟高
2. 简历文本过长
3. DeepSeek API 响应慢
4. 请求超时

**解决方案**：

```javascript
// 1. 检查网络延迟
const start = Date.now();
fetch('https://api.deepseek.com')
  .then(() => {
    const latency = Date.now() - start;
    console.log('网络延迟:', latency, 'ms');
  });

// 2. 限制简历文本长度
const maxLength = 5000; // 限制为 5000 字符
const truncatedText = text.substring(0, maxLength);

// 3. 添加超时处理
const timeout = 15000; // 15 秒超时
const timeoutPromise = new Promise((_, reject) => {
  setTimeout(() => reject(new Error('请求超时')), timeout);
});

Promise.race([
  parseResumeWithAI(text),
  timeoutPromise
]).catch(error => {
  console.log('超时，使用正则解析');
  return parseResumeWithRegex(text);
});
```

**优化建议**：
- 使用更快的网络
- 简历控制在 2 页以内
- 考虑使用正则解析（即时响应）

### 问题 10：成本超出预期

**症状**：
- DeepSeek API 费用比预期高

**可能原因**：
1. 重复解析同一份简历
2. 简历文本过长
3. 频繁测试

**解决方案**：

```javascript
// 1. 添加解析缓存
const cache = {};

async function parseResumeWithCache(text) {
  const hash = simpleHash(text);
  
  if (cache[hash]) {
    console.log('使用缓存结果');
    return cache[hash];
  }
  
  const result = await parseResumeWithAI(text);
  cache[hash] = result;
  return result;
}

function simpleHash(str) {
  let hash = 0;
  for (let i = 0; i < str.length; i++) {
    hash = ((hash << 5) - hash) + str.charCodeAt(i);
    hash = hash & hash;
  }
  return hash.toString();
}

// 2. 限制文本长度
const maxTokens = 2000; // 约 1500 字
const truncatedText = text.substring(0, maxTokens * 0.75);

// 3. 监控使用量
let apiCallCount = 0;
let totalCost = 0;

function trackApiCall(inputTokens, outputTokens) {
  apiCallCount++;
  const cost = (inputTokens * 0.001 + outputTokens * 0.002) / 1000;
  totalCost += cost;
  console.log(`API 调用次数: ${apiCallCount}, 总成本: ¥${totalCost.toFixed(4)}`);
}
```

**成本控制建议**：
- 同一份简历不重复解析
- 微调信息直接手动编辑
- 定期检查 DeepSeek 账户余额

## 🔄 应急方案

### 方案 1：回退到 V5

如果 V6 完全无法使用：

1. 编辑 `popup-v4-local.html`
2. 将 `<script src="popup-v6-ai.js"></script>` 改为 `<script src="popup-v5-robust.js"></script>`
3. 重新加载扩展

### 方案 2：使用正则解析

不配置 API Key，V6 会自动使用正则解析（与 V5 相同）。

### 方案 3：手动填写

如果自动解析都失败，使用"手动填写"标签页。
