# V6.1 更新 - 多段工作经验支持

## 🎉 新增功能

V6.1 版本新增了**多段工作经验解析**功能，能够智能识别并分段保存简历中的多个工作经历或实习经历。

## 📊 数据结构变化

### 旧版本（V6.0）

## 🔄 自动填充增强

### 新增字段匹配

```javascript
// 公司名称匹配
if (fieldName.includes('company') || fieldName.includes('公司')) {
  return resume.workExperiences[0].company;
}

// 职位匹配
if (fieldName.includes('position') || fieldName.includes('职位')) {
  return resume.workExperiences[0].position;
}

// 工作经验匹配（多段合并）
if (fieldName.includes('experience') || fieldName.includes('工作经验')) {
  return resume.workExperiences.map(exp => {
    let text = '';
    if (exp.company) text += exp.company;
    if (exp.position) text += ` - ${exp.position}`;
    if (exp.period) text += ` (${exp.period})`;
    if (exp.description) text += `\n${exp.description}`;
    return text;
  }).join('\n\n');
}
```

### 填充效果

当招聘网站有多个工作经验输入框时：
- 第一个框：填充第一段经历
- 第二个框：填充第二段经历
- 单个大文本框：填充所有经历（用空行分隔）

## 🔧 技术实现

### 数据结构定义

```typescript
interface WorkExperience {
  company: string;      // 公司名称
  position: string;     // 职位名称
  period: string;       // 时间段
  description: string;  // 工作描述
}

interface ResumeData {
  // ... 其他字段
  workExperiences: WorkExperience[];  // 多段工作经验（新增）
  workExperience: string;             // 合并文本（兼容旧版本）
}
```


## 💡 使用建议

### 给用户的建议

1. **简历格式建议**
   - 每段经历单独成段
   - 使用序号或分隔符（如空行）
   - 包含公司名、职位、时间、描述

2. **最佳格式示例**
   ```
   工作经验：
   
   1. 公司A - 职位A (2023.01-2024.01)
      工作描述...
   
   2. 公司B - 职位B (2022.01-2022.12)
      工作描述...
   ```

3. **避免的格式**
   - 所有经历混在一段
   - 没有明确的分隔符
   - 缺少公司名或职位

### 给开发者的建议

1. **扩展性**
   - 可以添加更多字段（如行业、公司规模等）
   - 可以支持项目经历、志愿者经历等

2. **优化方向**
   - 添加时间段解析（如"2年"、"6个月"）
   - 支持在职/离职状态识别
   - 添加工作成果量化提取
